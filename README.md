## WIP / private projects

<details>
<summary><strong>SparseHalide</strong> — hierarchical sparse data structures for Halide</summary>
<br/>
A <strong>C++20 header-only</strong> library that brings <strong>Taichi-style hierarchical sparse data structures (SNode trees)</strong> to the Halide compute framework.

Define a sparse hierarchy once with a fluent builder API. SparseHalide generates the appropriate Halide <code>Expr</code> index chains, buffer bindings, and scatter kernels automatically — so you can focus on simulation and compute logic instead of plumbing.
</details>

<details>
<summary><strong>Alwan (ألوان)</strong> — dependency-free colour science in pure C11</summary>
<br/>
A small, dependency-free <strong>colour science</strong> library in pure <strong>C11</strong>, built for precise and deterministic colour transformations without the overhead of external dependencies.

Alwan provides the mathematical foundations: colour space conversions, chromatic adaptation, appearance models, and spectral operations.

It is a <strong>colour science library</strong>, <em>not</em> a colour management library. It does not handle ICC profiles, device characterization, rendering intents, or profile connection spaces. For ICC workflows, Alwan is intended to complement dedicated tooling such as LittleCMS.
</details>

<details>
<summary><strong>Nadir</strong> — thin graphics abstraction with explicit backend selection</summary>
<br/>
A thin, explicit graphics layer with build-time backend selection for <strong>DX11</strong>, <strong>DX12</strong>, and <strong>Vulkan</strong>.

The goal is to keep the abstraction surface small, predictable, and close to the underlying API.
</details>

<details>
<summary><strong>ImDataViz</strong> — data visualization for Dear ImGui</summary>
<br/>
A data visualization API for Dear ImGui that aims for a feature set close to <strong>Mathematica's data visualization framework</strong>, while staying immediate-mode and embeddable in native tooling.
</details>

<details>
<summary><strong>to_Halide</strong> — low-level code to Halide conversion</summary>
<br/>
A converter from <strong>{HLSL, x86, C, C++}</strong> to <strong>Halide</strong>, including scheduling ideas inspired by <strong>Dexter</strong> and <strong>Helium</strong>.

Focused on bridging low-level code and Halide-based optimization workflows.
</details>

<details>
<summary><strong>Unify-Prox</strong> — unifying proximal optimization APIs in C++</summary>
<br/>
A <strong>C++-only</strong> API that aims to unify the feature set and workflows explored in the <strong>ProxImaL</strong> and <strong>Delta-Prox</strong> research projects.
</details>

<details>
<summary><strong>Falak</strong> — ethical web crawling in modern C++</summary>
<br/>
An ethical web crawler written in <strong>C++20</strong> for long-running exploration of web content with configurable bounds, polite rate limiting, <code>robots.txt</code> compliance, and resumable crawling sessions.

This one is still exploratory and may or may not be released.
</details>

<details>
<summary><strong>Secrets</strong> — ...</summary>
<br/>
Lot of protoypes, early stages, R&D, ... Rendering Techniques, Image and Geometry processing, Math Solvers (Proximal, Iterative, ...), and tons of applications of Optimal Transport!.
</details>
